| v16.9 | 2026-03-31 | **Two-phase execution model — eliminates tool-call budget exhaustion as a failure mode.** Root cause of 2026-03-31 blank dashboard: a single continuous run of ~35–45 tool calls sits at the edge of the Claude budget limit. Any retry, stored result, or error recovery tips it over, leaving the HTML build with no budget. **Fix: the skill now runs in two explicit phases separated by a user-confirmed hard stop.** Phase 1 ("Sales Q2" trigger) runs Steps 0–5 only — all Snowflake queries + Python validation → writes DS_PATH → prints a data summary → STOPS. Budget: ~22 calls. Phase 2 ("Build dashboard" trigger) runs Steps 6–12 only — reads DS_PATH, builds HTML, delivers file. Budget: ~9 calls. Zero Snowflake queries in Phase 2. The hard stop between phases means: (1) the user always sees and can verify the data before spending build budget; (2) if Phase 1 hits a tool limit mid-query, "Sales Q2" retriggers it from checkpoint (PD #46) — no data is lost; (3) if Phase 2 ever fails (extremely unlikely at 9 calls), "Build dashboard" re-reads DS_PATH and rebuilds instantly. Three skill changes: Stage 0 trigger routing block added (runs before all Snowflake queries, routes Phase 2 triggers directly to Step 6); Phase 1 hard stop added after Step 5; "build dashboard", "build the dashboard", "generate dashboard", "refresh data" added as trigger phrases. v16.9 is a pure architectural change — no data model changes, no HTML changes, no new NNRs. |
